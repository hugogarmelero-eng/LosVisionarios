Configura bien Claude

La mayoría usa Claude como si fuera un buscador. Configurado bien, es un empleado que ya te conoce antes de que abras la boca.

Paso 01
¿Free o Pro?

El plan gratuito tiene límites de mensajes y no da acceso a los modelos más potentes. Para negocios, Pro se paga solo desde el primer cliente.

Free: válido para probar y tareas sencillas
Pro (~20$/mes): Sonnet + Opus, más contexto, memoria, Projects
Si facturas 300€/mes usando Claude, ya está más que amortizado

Módulo I  /  Cosas que debes conocer
Conocimiento esencial

Cosas que debes
conocer de Claude

No prompts. La información que necesitas para entender cómo funciona Claude por dentro y sacarle el máximo a cada conversación.

01
Claude vs ChatGPT: la diferencia real

Ambos son IAs, pero no son lo mismo. Claude destaca en razonamiento largo, análisis profundo y seguir instrucciones complejas sin perder el hilo.

Claude: mejor en textos largos, instrucciones complejas y análisis estratégico
ChatGPT: mejor ecosistema de plugins e integraciones externas
Para negocios y estrategia: Claude gana. Para coding rápido: empatan
02
Los modelos: Haiku, Sonnet, Opus

Tres versiones con distinto equilibrio entre velocidad, coste y potencia. Elegir bien cuál usar según la tarea te ahorra dinero y tiempo.

Haiku: rápido y barato. Automatizaciones en volumen y tareas simples
Sonnet: equilibrio perfecto. Úsalo para la mayoría de tareas de negocio
Opus: el más potente. Análisis complejos y decisiones importantes
03
Contexto y memoria: por qué "olvida"

Claude no tiene memoria permanente por defecto. Cada conversación es una hoja en blanco. Entender esto te evita frustraciones y te enseña a estructurar bien tus chats.

Cada chat tiene una ventana de contexto — todo lo que cabe en esa sesión
Al abrir un chat nuevo, Claude no recuerda el anterior (salvo memoria activada)
Solución: Projects para que tu contexto persista entre sesiones
04
Claude Code: para qué sirve

Claude Code es una herramienta de línea de comandos que ejecuta tareas directamente en tu ordenador. No necesitas saber programar para usarla bien.

Lee, edita y crea archivos en tu ordenador de forma autónoma
Úsalo para: procesar datos, crear herramientas internas, automatizar tareas locales
Como estratega: delega tareas técnicas sin depender de programadores

Módulo II  /  Introducción a n8n y Make
Base fundamental

Introducción a
n8n y Make

Antes de tocar ninguna herramienta, necesitas entender cómo viajan los datos. Cuando lo entiendas, todo lo demás tiene sentido solo.

1.1 — Base
Cómo viajan los datos

Toda automatización funciona igual: algo entra, se procesa, y algo sale. Este es el único concepto que necesitas para entender cualquier flujo.

Entrada (trigger): el evento que arranca el flujo. Un email, un formulario, una hora del día.
Procesamiento: lo que haces con esos datos. Filtrar, transformar, clasificar, enviar a una IA.
Salida (output): el resultado. Un email enviado, un registro en CRM, una notificación.
1.2 — Conceptos clave
Concepto 01
Webhooks

Un webhook es como un timbre: cuando algo ocurre en una app, te avisa en el momento. No tienes que ir a mirar — él te llama.

Una app externa envía datos a tu flujo cuando ocurre un evento
Ejemplo: alguien compra en tu tienda → Stripe hace un webhook → tu flujo arranca automáticamente
Son en tiempo real. No hay espera ni polling.
Concepto 02
APIs

Una API es el puente entre dos aplicaciones. Es lo que permite que tu flujo de n8n le hable a Google Sheets, a OpenAI, a tu CRM o a cualquier otra herramienta.

Piénsalo como un camarero: tú pides, él lleva la petición a cocina y trae el resultado
Ejemplo: tu flujo llama a la API de Claude → Claude responde → tu flujo usa esa respuesta
Casi toda herramienta moderna tiene API. Eso significa que casi todo se puede conectar.
Concepto 03
Credenciales (API Keys)

Para que tu flujo pueda hablarle a una app externa, necesita identificarse. Las credenciales son como el DNI de tu automatización.

Son tokens o claves que generan las apps (OpenAI Key, Airtable Token, etc.)
Seguridad básica: nunca las compartas, nunca las pongas en código público
n8n y Make tienen gestores de credenciales seguros — úsalos siempre
Si una clave se filtra → cámbiala inmediatamente desde el panel de la app
1.3 — Herramientas
n8n vs Make
¿Cuál usar?

No hay una respuesta universal. La elección depende del perfil del proyecto y del cliente. Aquí tienes la guía rápida.

n8n: máxima flexibilidad, lógica avanzada, se puede autoalojar. Ideal para agencias y proyectos complejos.
Make: más visual e intuitivo, perfecta para empezar y para clientes que quieren gestionar sus flujos.
Recomendación Hugo: aprende n8n para tu agencia, ofrece Make a clientes que quieren autonomía.
1.4 — Con IA vs Sin IA
Concepto clave
Automatización con IA vs sin IA

Esta distinción es lo que diferencia una agencia básica de una premium. Aquí está la diferencia real.

Sin IA: reglas fijas. Si X entonces Y. Funciona bien para procesos predecibles y repetitivos.
Con IA: decisiones dinámicas. El flujo analiza, razona y actúa según el contexto.
Ejemplo sin IA: si el cliente marca "urgente" → asignar a soporte premium.
Con IA: Claude lee el mensaje, detecta la urgencia real aunque no la diga explícitamente.
Módulo II  /  Tus primeras automatizaciones
Modo práctica

Tus primeras
automatizaciones

Ya entiendes la base. Ahora toca construir. Esta sección te lleva de cero a tu primer flujo funcionando — paso a paso, sin rodeos.

2.1 — Estructura
Los 4 elementos de toda automatización

Todo flujo, por complejo que sea, tiene exactamente estos 4 elementos. Entiéndelos y podrás construir cualquier cosa.

Trigger: lo que arranca el flujo. Formulario, email, horario, webhook.
Nodos: cada paso que hace algo. Filtrar, transformar, llamar a una API.
Lógica: las condiciones. Si X → haz esto. Si no → haz aquello.
Output: el resultado final. Mensaje enviado, dato guardado, tarea creada.
2.2 — Guía paso a paso
El proceso
Cómo construir tu primer flujo

Sigue este orden y no te saltará ningún paso. Es el mismo proceso que usa Hugo para construir flujos para clientes.

1. Elige un problema real — no automatices por automatizar. Debe haber un dolor claro.
2. Define el trigger — ¿qué arranca el flujo? ¿Cuándo debe ejecutarse?
3. Elige las herramientas — ¿qué apps necesitan hablar entre sí?
4. Construye el flujo — empieza simple. Un trigger, un nodo, un output.
5. Testea — ejecuta con datos reales. Revisa cada nodo.
6. Optimiza — añade lógica, manejo de errores y guardrails.
2.3 — Errores comunes
Problema mal definido
Automatizas antes de entender qué problema exacto resuelves. Resultado: flujo que no sirve para nada.
Flujo demasiado complejo
Empezar con 20 nodos cuando con 4 es suficiente. Empieza simple, añade complejidad después.
No testear con datos reales
Los flujos que fallan en producción son los que solo se testearon con datos de prueba perfectos.
2.4 — Ciberseguridad
MUY IMPORTANTE
Protege tus flujos y los datos de tus clientes

Cuando automatizas para clientes manejas sus datos. Un error aquí puede costarte un cliente, una demanda o tu reputación.

Credenciales: usa siempre el gestor de n8n/Make. Nunca en código en texto plano.
Datos sensibles: no pases datos personales innecesarios entre nodos. Principio de mínimo acceso.
Webhooks: activa verificación de firma cuando la app lo permita. Evita peticiones falsas.
Logs: activa el registro de errores para detectar comportamientos extraños.
2.5 — Guardrails
Control de flujos
Guardrails: el cinturón de seguridad

Un guardrail es una barrera que evita que tu flujo haga cosas que no debería. Imprescindibles cuando automatizas para clientes reales.

Validación de datos: comprueba que lo que entra tiene el formato correcto antes de procesarlo.
Límites de ejecución: máximo de llamadas por hora/día para evitar costes disparados o baneos.
Filtros antes de acciones críticas: antes de enviar un email o borrar un dato, un nodo que confirma que todo está bien.
Manejo de errores: si un nodo falla, el flujo no debe morir silencioso. Que te avise.

3.2 — Ventas
Área Ventas
Pipeline de ventas automatizado

El 80% de las ventas requieren más de 5 seguimientos. El 90% de los vendedores abandona antes del 3º. Un flujo automatizado no se cansa.

Flujo: Lead en CRM → Claude evalúa temperatura → secuencia de seguimiento personalizada → alerta cuando está listo para cerrar
Resultado esperado: ciclo de venta -30%. Seguimientos sin intervención manual.
Herramientas: HubSpot/Pipedrive + n8n + Claude + Gmail
3.3 — Operaciones
Área Operaciones
Sincronización y tareas internas

Las empresas pierden horas semanales en copiar datos entre herramientas, crear reportes manuales y coordinar tareas repetitivas. Todo esto es automatizable.

Resultado esperado: 5-10 horas semanales ahorradas por empresa. ROI inmediato y medible.

Resultado: clientes más satisfechos, más testimonios, menos tiempo manual de entrega.

Módulo III  /  Tu primer cliente
Acción desde hoy

Cómo conseguir
tu primer cliente

El primer cliente no llega por tener la web perfecta ni la marca bonita. Llega cuando paras de prepararte y empiezas a hablar con gente real. Aquí tienes todo lo que necesitas.

Base
La mentalidad correcta

El 90% se queda en la fase de preparación indefinidamente. La web no está, el logo no es perfecto, el nicho no está claro. Eso no importa todavía.

Acción imperfecta > preparación perfecta. Un cliente real te enseña más que 100 horas de cursos.
No necesitas web, ni marca, ni logo para conseguir tu primer cliente. Necesitas una conversación.
Valida con personas reales cuanto antes. El mercado siempre tiene la última palabra.
Objetivo del primer cliente: aprender, no ganar millones. Luego escala.
Canales de captación
DMs y WhatsApp
El canal más directo. Mensaje corto, claro y personalizado. No copies y pegues el mismo texto a todos.
Negocios locales
Entra por la puerta o llama. Restaurantes, clínicas, academias. Tienen problemas reales y pagan bien si les resuelves algo concreto.
Plataformas freelance
Fiverr, Upwork, Freelancer. Ideales para conseguir los primeros proyectos con reseñas. La competencia es alta pero el volumen también.
Networking
Tu entorno más cercano es la mina de oro más ignorada. Amigos, familia, conocidos. Cuéntales a qué te dedicas.
Contenido en redes
No necesitas 10.000 seguidores. Un vídeo explicando cómo automatizaste algo real puede traerte 3 consultas esta semana.
Cold outreach
Email frío bien planteado: personalizado, corto, con un problema específico del receptor. Tasa de respuesta del 10-20% si se hace bien.
Qué ofrecer
Simple, específico y con resultado claro

No vayas con "te ayudo con IA". Nadie compra eso. Ve con un problema concreto que resuelves y un resultado medible.

MAL: "Ofrezco servicios de IA y automatización para tu negocio"
BIEN: "Automatizo el seguimiento de leads para que no pierdas ninguna venta por falta de contacto"
Al inicio: una sola automatización, bien hecha, que resuelva un dolor real. No intentes vender todo.
El cliente no compra tecnología, compra tiempo ahorrado, ventas ganadas o problemas resueltos.
El proceso
Contacto → videollamada → cierre

Este es el único proceso que necesitas para cerrar clientes. No lo compliques. La videollamada es el punto de inflexión: sin ella, sin cierre.

Contacto: mensaje corto, personalizado, con un problema específico del receptor
Interés: si responde, no vendas. Haz preguntas. Entiende su situación.
Videollamada: aquí diagnosticas el problema real. Nunca envíes propuesta sin esto.
Cierre: propuesta basada en lo que escuchaste, no en lo que tú querías vender.

Módulo III  /  Modelo de monetización
Estructura tu oferta

Modelo de monetización
y packs de servicios

Sin estructura de oferta clara, cada cliente es una negociación desde cero. Con packs definidos, vendes más rápido, a mayor ticket y con menos fricción.

Por qué estructurar
Packs vs precio por hora

Vender horas limita tu crecimiento. Cuando vendes un sistema o resultado, tu capacidad de facturar no depende de cuánto tiempo trabajas.

Por hora: techo de ingresos = horas disponibles. Trabajas más → ganas más. No escala.
Por pack/resultado: el valor es el sistema que entregas, no el tiempo que tardas.
Los packs facilitan la decisión del cliente, suben el ticket medio y evitan clientes que piden infinito.
Los 3 packs
Pack Básico
Entrada accesible y primera victoria

Diseñado para clientes que quieren probar sin comprometerse a mucho. Sirve para entrar, demostrar valor y abrir la puerta al upsell.

1–2 automatizaciones simples y concretas
Problema específico y resultado claro
Soporte básico (email, 30 días)
Precio orientativo: 500 – 1.500€
Ejemplo: automatizar el seguimiento de leads de un formulario web
Pack Intermedio
El más vendido — el punto dulce

Este es el pack que cierra solo. Suficiente valor para que el cliente lo vea claro y suficiente margen para que tú lo ejecutes bien.

3–5 automatizaciones con lógica avanzada
Integración con CRM, email marketing o herramientas del cliente
Soporte activo 60 días (llamadas incluidas)
Documentación del flujo entregada
Precio orientativo: 1.500 – 3.500€
Pack Premium
Sistema completo + relación continua

Para clientes que quieren delegar la transformación digital completa. Aquí entras como socio estratégico, no como proveedor.

Diagnóstico completo + implementación total
+5 automatizaciones con IA integrada

Mantenimiento mensual recurrente
Precio orientativo: 3.500€+ / mensualidad recurrente

Fase 1
El Diagnóstico — se cobra primero

Uno de los errores más comunes: hacer trabajo estratégico gratis pensando que "así consigo el cliente". El diagnóstico tiene valor. Cóbralo.

Si alguien no paga por entender su problema → no pagará por resolverlo
El diagnóstico incluye: análisis del negocio, mapeo de procesos, identificación de ineficiencias, detección de oportunidades de automatización, revisión de herramientas, evaluación de ciberseguridad y diseño de roadmap
Se entrega en Notion: visual, claro y accionable. Es un documento de valor, no un PowerPoint genérico
Precio orientativo del diagnóstico: 300–800€. Se descuenta si contratan la implementación.
Fase 2
Implementación — ejecutar lo definido

Una vez pagado el diagnóstico, el cliente ya ha validado que confía en ti. Ahora ejecutas exactamente lo que definiste en el roadmap.

Se construyen las automatizaciones definidas en el diagnóstico
Integraciones entre herramientas del cliente
Testing exhaustivo antes de entregar
Optimización y ajustes post-entrega
Ventaja del modelo en 2 fases: el cliente ya está comprometido. El cierre es casi automático.

Módulo III  /  Temas legales
Protégete desde el inicio

Temas legales
que debes conocer

No necesitas ser abogado. Pero sí necesitas saber lo mínimo para operar con seguridad, cobrar sin problemas y protegerte ante cualquier conflicto con un cliente.

Estructura legal
Autónomo vs Sociedad

La estructura legal depende de tu país, tu facturación y tus objetivos. Lo importante es no operar en negro — te expone a riesgos innecesarios.

Al inicio: lo más sencillo es darte de alta como autónomo o equivalente en tu país
Al crecer: una sociedad limitada (SL) protege tu patrimonio personal y da más credibilidad a clientes grandes
Recomendación: consulta con un asesor fiscal local antes de decidir. El coste de la consulta siempre vale la pena.
Con estructura legal puedes emitir facturas, deducir gastos y operar con total tranquilidad.
Los 2 contratos
Contratos
Siempre 2 contratos separados

Un contrato por el diagnóstico y otro por la implementación. Separados porque son dos servicios distintos con precios y alcances distintos.

Contrato 1 — Diagnóstico: alcance del análisis, entregable (informe Notion), precio y condiciones de pago
Contrato 2 — Implementación: lista de automatizaciones a construir, plazos, precio total y forma de pago
MUY IMPORTANTE: nunca garantices KPIs específicos (más ventas, más clientes). Solo garantizas el proceso y la entrega técnica.
Si el cliente quiere resultados garantizados → es otro modelo de negocio (comisión por performance). Distinto contrato, distinto precio.
⚠️ Importante — Lo que no debes prometer
NUNCA garantices resultados de negocio específicos en el contrato: ✕ "Aumentaremos tus ventas un 30%" ✕ "Conseguirás 50 leads al mes" ✕ "Reducirás costes en X€" Lo que SÍ puedes garantizar: ✓ La entrega técnica de las automatizaciones definidas ✓ El informe de diagnóstico según el alcance acordado ✓ El funcionamiento correcto de los flujos implementados Si un cliente quiere resultados garantizados → cambia el modelo a comisión por performance. Es otro tipo de contrato, con mucho más riesgo para ti. Asesórate antes.

Genera una propuesta comercial para un cliente con este contexto: - Sector: [SECTOR] - Problema principal: [PROBLEMA] - Automatizaciones a implementar: [LISTA] - Herramientas a integrar: [HERRAMIENTAS] - Inversión objetivo: [PRECIO €] La propuesta debe incluir: 1. Email frío que consigue respuesta

Cuándo usarlo: cuando quieres contactar a una empresa que no te conoce. El secreto: personalización real + problema específico + CTA claro.

📋 Prompt
Escribe un email frío para contactar a [NOMBRE/EMPRESA] del sector [SECTOR]. Lo que sé de ellos: [2-3 datos específicos: tamaño, producto, algo que vi en su web o redes] Mi servicio: automatizo procesos de [ÁREA] para que [RESULTADO CONCRETO]. El email debe: - Tener menos de 100 palabras - Empezar con algo específico de ellos (no genérico) - Mencionar un problema concreto que probablemente tienen - Proponer una videollamada de 20 minutos sin presión - Tener un asunto que invite a abrir No uses frases típicas de vendedor. Que suene como un profesional que ha investigado, no como spam.

Automatizaciones con IA
Automatización · Diseño de flujo
Diseñar una automatización desde cero

Cuándo usarlo: cuando tienes claro el problema a resolver pero no sabes exactamente cómo estructurar el flujo. Claude lo diseña y tú lo construyes.

📋 Prompt
Necesito diseñar una automatización para resolver este problema: Negocio: [TIPO DE NEGOCIO] Problema actual: [DESCRIPCIÓN DEL PROBLEMA MANUAL O INEFICIENCIA] Herramientas que usa el cliente: [LISTA DE APPS] Herramientas disponibles para automatizar: n8n / Make / [OTRAS] Diseña el flujo completo con: 1. Trigger: qué arranca la automatización y cuándo 2. Pasos del flujo en orden (nodo por nodo) 3. Lógica condicional si la hay (IF/ELSE) 4. Output final: qué ocurre al terminar el flujo 5. Posibles errores a anticipar y cómo manejarlos 6. Estimación de tiempo de construcción Sé específico con los nombres de nodos y apps reales.
Automatización · Prompt de sistema IA
Crear el prompt de sistema para un agente

Cuándo usarlo: cuando necesitas configurar un nodo de Claude dentro de n8n o Make y tienes que escribir el prompt de sistema que define cómo se comporta la IA en ese flujo.

📋 Prompt
Necesito escribir el prompt de sistema para un nodo de IA dentro de una automatización. Contexto del flujo: - Qué hace el flujo: [DESCRIPCIÓN] - Qué datos recibe el nodo de IA: [TIPO DE INPUT] - Qué debe hacer la IA con esos datos: [TAREA ESPECÍFICA] - Qué debe devolver: [FORMATO DEL OUTPUT] Restricciones importantes: - [LIMITACIONES O REGLAS DE NEGOCIO] Escribe un prompt de sistema claro, preciso y sin ambigüedades. Que la IA sepa exactamente qué hacer con cualquier input que reciba. Incluye un ejemplo de input y el output esperado.
